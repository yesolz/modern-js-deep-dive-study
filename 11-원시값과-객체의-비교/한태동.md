# 11장

### 1. 원시 타입 vs 객체 타입 차이점

- **값의 특성**: 원시 타입은 변경 불가능한 값(불변성)이며, 객체 타입은 변경 가능한 값(가변성)이다.
- **변수에 할당 시**: 원시 타입은 변수에 실제 값이 저장되며, 객체 타입은 참조 값(메모리 주소)이 저장된다.
- **다른 변수에 할당 시**: 원시 값은 복사되어 전달(값에 의한 전달)되고, 객체는 참조 값이 복사되어 전달(참조에 의한 전달)된다.

### 2. 원시 값

### 2.1. **변경 불가능한 값 (불변성)**

- 원시 값은 읽기 전용이며, 한번 생성된 값은 변경할 수 없다. 새로운 값을 재할당하면 기존 메모리 공간이 아닌 새로운 메모리 공간에 값이 저장된다.
- 불변성 덕분에 데이터의 신뢰성이 높아지며, 값의 변경을 추적하기 쉽다.

### 2.2. **문자열과 불변성**

- 문자열은 원시 타입으로 변경 불가능하며, 새로운 문자열이 생성되면 기존 문자열을 수정하는 것이 아니라 새로운 메모리 공간에 저장된다.

### 2.3. **값에 의한 전달**

- 원시 값을 가진 변수를 다른 변수에 할당할 때 값이 복사되어 전달되며, 두 변수는 별개의 메모리 공간을 사용한다. 따라서 한 변수의 값이 변경되어도 다른 변수에 영향을 미치지 않는다.

### 3. 객체

### 3.1. **변경 가능한 값 (가변성)**

- 객체는 원시 값과 달리 변경이 가능하다. 객체를 할당한 변수는 참조 값을 통해 메모리 내 객체를 가리키며, 재할당 없이 프로퍼티를 동적으로 추가, 수정, 삭제할 수 있다.

### 3.2. **참조에 의한 전달**

- 객체를 다른 변수에 할당하면 참조 값이 복사되어 전달된다. 이로 인해 여러 변수가 동일한 객체를 가리킬 수 있으며, 어느 한쪽에서 객체를 변경하면 다른 변수도 영향을 받는다.
- 참조에 의한 전달은 유연성을 제공하지만, 원본 객체의 불변성이 보장되지 않아 예기치 않은 값의 변경을 추적하기 어려운 단점이 있다.

### 4. 얕은 복사 vs 깊은 복사

- **얕은 복사**: 객체의 참조 값만 복사하여 중첩된 객체는 동일한 참조 값을 공유한다.
- **깊은 복사**: 중첩된 객체까지 모두 복사하여 완전히 독립적인 객체를 생성한다.

이러한 차이는 자바스크립트의 원시 값과 객체의 동작 방식을 이해하는 데 중요한 개념이며, 값의 불변성과 참조의 전달 방식을 명확하게 구분하는 것이 필요합니다.
